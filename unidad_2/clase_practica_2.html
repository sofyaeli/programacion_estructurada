Clase Práctica No. 2

Objetivos

Desarrollar un programa en C# que permita gestionar el cambio de una clave de usuario mediante la validación de datos ingresados por teclado. Utilizando estructuras de control condicionales y repetitivas, el programa verificará que la nueva clave sea confirmada correctamente por el usuario. Esta actividad busca fortalecer el manejo de variables, captura de datos, toma de decisiones y repetición de procesos, contribuyendo además al trabajo en equipo y al desarrollo del pensamiento lógico.

Descripción del Problema

Se requiere elaborar un programa que solicite al usuario su nombre y una clave inicial. Posteriormente, el sistema informará que la clave ha caducado y solicitará una nueva clave.

Una vez ingresada la nueva clave, el usuario deberá confirmarla. Si ambas coinciden, se mostrará el mensaje: 

La clave se ha cambiado con éxito.

Si las claves no coinciden, el sistema solicitará nuevamente la confirmación de la clave, permitiendo un máximo de cinco intentos. Si al finalizar los intentos la confirmación sigue siendo incorrecta, se mostrará el mensaje: 

Clave Incorrecta. Usuario Bloqueado.

Algoritmo

  • Inicio.
  • Solicitar el nombre del usuario.
  • Solicitar la clave actual.
  • Mostrar mensaje de clave caducada.
  • Solicitar una nueva clave.
  • Solicitar la confirmación de la nueva clave.
  • Comparar ambas claves.
  • Si coinciden, mostrar mensaje de éxito.
  • Si no coinciden, permitir hasta cinco intentos.
  • Si supera los intentos permitidos, bloquear al usuario.
  • Fin.

Código en C#

using System;

class Program
{
    static void Main()
    {
        string nombre;
        string claveActual;
        string nuevaClave;
        string confirmacion;

        bool claveCorrecta = false;
        int intentos = 0;

        Console.Write("Ingrese su nombre: ");
        nombre = Console.ReadLine();

        Console.Write("Ingrese su clave actual: ");
        claveActual = Console.ReadLine();

        Console.WriteLine(
            "\nSu clave ha caducado. Por favor, introduzca su nueva clave."
        );

        nuevaClave = Console.ReadLine();

        while (intentos < 5)
        {
            Console.WriteLine(
                "Por favor, confirme su nueva clave:"
            );

            confirmacion = Console.ReadLine();

            if (nuevaClave == confirmacion)
            {
                claveCorrecta = true;
                break;
            }

            intentos++;

            if (intentos < 5)
            {
                Console.WriteLine(
                    "Por favor, vuelva a introducir su clave."
                );
            }
        }

        if (claveCorrecta)
        {
            Console.WriteLine(
                "\nLa clave se ha cambiado con éxito."
            );
        }
        else
        {
            Console.WriteLine(
                "\nClave Incorrecta. Usuario Bloqueado."
            );
        }
    }
}

Ejemplo de Ejecución

  • Nombre: Sofia
  • Clave actual: 1234
  • Nueva clave: abc123
  • Confirmación: abc123
  • Resultado: La clave se ha cambiado con éxito.

Análisis de Resultados

El programa utiliza una estructura repetitiva while para permitir hasta cinco intentos de confirmación de la nueva clave. Además, emplea una estructura condicional if para verificar si las claves coinciden y tomar la decisión correspondiente.

Conceptos Aplicados

  • Entrada y salida de datos.
  • Estructuras condicionales (if).
  • Estructuras repetitivas (while).
  • Validación de datos.
  • Control de intentos.
  • Comparación de cadenas de texto.

Conclusiones

  • El uso del ciclo while permitió controlar el número máximo de intentos de confirmación.
  • La estructura if facilitó la toma de decisiones según las respuestas proporcionadas por el usuario.
  • Se reforzó el manejo de variables de texto y la comparación de cadenas en C#.
  • La actividad permitió comprender cómo funcionan los procesos de autenticación y validación presentes en muchas aplicaciones reales.
