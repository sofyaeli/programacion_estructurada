Unidad 3: Programación Orientada a Objetos en C#

Objetivo

Comprender los fundamentos de la Programación Orientada a Objetos (POO) en C#, incluyendo la creación de clases y objetos, el uso de propiedades y métodos, así como los principios de herencia, polimorfismo, encapsulamiento e interfaces. Estos conceptos permiten desarrollar aplicaciones más organizadas, reutilizables y fáciles de mantener.

Introducción

La Programación Orientada a Objetos (POO) es un paradigma de programación basado en la creación de objetos que representan entidades del mundo real. Cada objeto posee características (atributos) y comportamientos (métodos), permitiendo modelar problemas de manera más intuitiva y eficiente.

3.1 Clases y Objetos

Una clase es una plantilla o modelo que define las características y comportamientos que tendrán los objetos.

Un objeto es una instancia de una clase, es decir, una representación concreta creada a partir de dicha plantilla.

Ejemplo de Clase

class Persona
{
    public string Nombre;
    public int Edad;
}

Creación de Objetos

Persona persona1 = new Persona();

persona1.Nombre = "Juan";
persona1.Edad = 20;

En este ejemplo, Persona es la clase y persona1 es un objeto creado a partir de ella.

Ventajas de Utilizar Clases y Objetos

  • Permiten organizar mejor el código.
  • Facilitan la reutilización de componentes.
  • Representan entidades del mundo real.
  • Mejoran el mantenimiento de las aplicaciones.

3.2 Propiedades y Métodos

Las propiedades permiten almacenar y gestionar información dentro de una clase.

Los métodos representan acciones o tareas que un objeto puede realizar.

Ejemplo

class Persona
{
    public string Nombre { get; set; }

    public void Saludar()
    {
        Console.WriteLine("Hola, soy " + Nombre);
    }
}

La propiedad Nombre almacena información del objeto, mientras que el método Saludar() define un comportamiento.

Uso del Método

Persona persona1 = new Persona();

persona1.Nombre = "Ana";
persona1.Saludar();

3.3 Herencia, Polimorfismo y Encapsulamiento

Encapsulamiento

El encapsulamiento consiste en proteger los datos de una clase y controlar su acceso mediante propiedades y métodos. 

class Cuenta
{
    private double saldo;

    public void Depositar(double monto)
    {
        saldo += monto;
    }
}

La variable saldo es privada y únicamente puede modificarse mediante métodos autorizados.

Herencia

La herencia permite crear nuevas clases basadas en clases existentes, reutilizando atributos y métodos. 

class Animal
{
    public void Comer()
    {
        Console.WriteLine("El animal está comiendo");
    }
}

class Perro : Animal
{
}

La clase Perro hereda el método Comer() de la clase Animal.

Polimorfismo

El polimorfismo permite que un mismo método tenga comportamientos diferentes dependiendo del objeto que lo utilice. 

class Animal
{
    public virtual void Sonido()
    {
        Console.WriteLine("Sonido genérico");
    }
}

class Perro : Animal
{
    public override void Sonido()
    {
        Console.WriteLine("Guau Guau");
    }
}

Aunque ambos utilizan el método Sonido(), cada clase genera un comportamiento distinto.

Beneficios de la Herencia, Encapsulamiento y Polimorfismo

  • Favorecen la reutilización de código.
  • Mejoran la seguridad de los datos.
  • Facilitan el mantenimiento del software.
  • Permiten desarrollar sistemas escalables.

3.4 Interfaces

Una interfaz define un conjunto de métodos que una clase debe implementar. Actúa como un contrato que garantiza ciertas funcionalidades.

Definición de una Interfaz

interface IVehiculo
{
    void Arrancar();
}

Implementación

class Auto : IVehiculo
{
    public void Arrancar()
    {
        Console.WriteLine("El auto ha arrancado");
    }
}

La clase Auto está obligada a implementar el método Arrancar() definido en la interfaz.

Ventajas de las Interfaces

  • Promueven la flexibilidad del diseño.
  • Favorecen el desacoplamiento entre clases.
  • Facilitan el mantenimiento y las pruebas.
  • Permiten implementar múltiples comportamientos.

Aplicaciones Prácticas

  • Sistemas de gestión empresarial.
  • Aplicaciones bancarias.
  • Videojuegos.
  • Sistemas educativos.
  • Aplicaciones móviles y web.

Conclusiones

  • Las clases y objetos constituyen la base de la Programación Orientada a Objetos.
  • Las propiedades y métodos permiten definir características y comportamientos de los objetos.
  • El encapsulamiento protege los datos y mejora la seguridad del software.
  • La herencia favorece la reutilización de código y la creación de jerarquías de clases.
  • El polimorfismo permite adaptar comportamientos según el contexto.
  • Las interfaces facilitan el diseño de aplicaciones flexibles y escalables.
  • La Programación Orientada a Objetos es uno de los pilares fundamentales para el desarrollo profesional de software moderno.
